## 背景：前任者モデルの評価

前任者は Faster R-CNN（ResNet-50 FPN）でブラジルアマゾンの違法滑走路178枚（Planet NICFI画像）を学習したモデルを残した。分析の結果、以下の課題を確認した。

| 課題 | 内容 |
|---|---|
| データ量不足 | 178枚は物体検出として極めて少ない |
| 検証セットの独立性なし | train/valが同一プールからのランダム分割で、未知データへの性能が未検証 |
| 地理的偏り | ブラジルのみ。ペルー他地域への汎化性能は不明 |
| 信頼度閾値なし | 推論時に全検出を出力、誤検知のフィルタリングなし |
| NIRバンド未使用 | 4バンド画像をRGBに変換し、植生検出に有効な近赤外を捨てている |
| 訓練設定の不一致 | コードとレポートでepoch/lrが異なり再現性が低い |
| mAP数値の欠如 | 定量評価が実質されていない |

→ 結論：**モデルは出発点として使えるが、現状のまま実運用には使えない。** 最大の資産は前任者が築いたグラウンドトゥルース構築のノウハウ（複数データソースの統合・フィルタリングルール）。

## データセット拡充の方針

### 検討した選択肢と却下理由

| 選択肢 | 却下理由 |
|---|---|
| 新規データセットをゼロから探す | 信頼できる第三者データソースが乏しく、現実的でない |
| ランダムサンプリングで全数確認 | 陽性率が0.01%未満で、スタッフ工数に見合わない |
| 変化検出のみで運用 | 既存（伐採済みで久しい）滑走路の定期再確認ロジックと相性が悪く、3ヶ月/6ヶ月のステータス遷移が機能しない |

### 採用する方針：Active Learning（人間参加型の段階的データ拡充）

```
既存Faster-RCNNモデルで検出（閾値なし）→ スタッフ確認 500枚
＋
NDVI変化検出（細長形状フィルタ）で候補抽出 → スタッフ確認 500枚
＋
元178枚（陽性のみ）
─────────────────────────────
計 約1178枚 を YOLO で再学習
```

**なぜ2ソースを混ぜるか：**
- 既存モデル検出 → モデルが既に得意なパターンの精緻化、誤検知（陰性）を学習データとして活用
- 変化検出 → モデルが見逃しやすい新規・非典型パターンの発掘（False Negative対策）

**陰性サンプルの価値：** Faster R-CNN/YOLOとも、明示的な陰性（紛らわしい非滑走路：農道・川岸・農地境界など）を学習データに加えることで誤検知が減る。前任者の178枚は全て陽性のみで、紛らわしい陰性パターンを学習していない点が誤検知の一因と推測される。

**再学習サイクル：** 数年に一度（3〜10年スパン）、同様のプロセスを繰り返す。滑走路建設手法の変化や衛星画像の解像度向上に追従するため。

## 2画像ソースの役割分担（Sentinel-2 と Planet NICFI）

当初案（ARCHITECTURE.md記載）は両ソースにそれぞれ独立したYOLOモデルを学習させる想定だった。検討の結果、以下に変更する。

### 変更の理由

Sentinel-2は解像度10mで、滑走路の幅（15〜30m）が1〜3ピクセルにしかならず、**形状による独立判定が困難**。一方Planet NICFIは4.77mで前任者データもこちらに統一されているため、形状認識を伴う本検出はNICFIに一本化する。

### 確定した役割分担

| ソース | 役割 | 頻度 | 確定力 |
|---|---|---|---|
| **Planet NICFI** | 本検出（YOLO推論、形状判定） | 月次・ペルー全土 | `active`への格上げを担う唯一のソース |
| **Sentinel-2** | 早期警戒（変化検出のみ、形状判定なし） | 週次・晴天域のみ | `unconfirmed`登録のみ、確定はしない |

### ステータス遷移ロジック（確定）

優先順位：active → unconfirmed → inactive の順に判定。

```
active:      NICFI検出が3ヶ月以内
unconfirmed: (NICFI検出が3〜6ヶ月前) OR (Sentinel検出が6ヶ月以内)
inactive:    NICFI検出が6ヶ月超 AND Sentinel検出も6ヶ月超
```

**具体例（7/1 NICFI画像に滑走路なし、7/8新規開設、Sentinelが7/8に変化検出）：**
1. 7/8 Sentinel変化検出 → 即座に `unconfirmed`（黄色）として登録
2. 8/1 NICFI月次更新で確認 → `active`（緑）に格上げ
3. 仮にNICFIで6ヶ月確認されなければ自動的に `inactive`（Sentinel単独の誤検知だった可能性として除外）

**Sentinel単独検出時の信頼度：** 固定値ではなく、検出ごとの変動信頼度を採用（詳細は下記「構築フロー」参照）。NDVI変化量等の連続値の特徴量から、較正曲線を用いて検出ごとに信頼度を算出する。

## Sentinel-2変化検出の出力形式と候補フィルタ（確定）

**出力形式：座標点＋スコア**（bboxは採用しない）。Sentinel-2の10m解像度では滑走路幅（1〜3ピクセル）の形状情報が得られず、bboxを出しても「NDVI変化のあった連結ピクセル塊の外接矩形」に過ぎず精度向上に寄与しないため。重複マッチング（500m近接判定）にも点座標で十分。

**リアルタイム性の確保：** 複数回連続確認をゲート条件にはしない。クリア画像が得られた回ごとに即座に`unconfirmed`候補として登録し、再検出のたびに信頼度を積み増す（既存のduplicate detection logicのconfidence更新で実現、ARCHITECTURE.md参照）。これにより最短5日サイクルでの早期警戒を維持しつつ、単発ノイズ（雲影・水面反射等）は再検出されないため自然に信頼度が上がらず淘汰される。

**候補フィルタ条件（確定）：**
- NDVI変化検出（伐開・植生消失）
- 細長形状フィルタ（アスペクト比）
- **直線性フィルタ**（追加） — 滑走路は直線に近い。道路は蛇行、川は湾曲、伐採フロントは不定形なため区別に有効
- **最小長さ閾値**（追加） — 一般的な小規模伐採・農地区画を除外

→ 「孤立度（既存道路・河川からの距離）」「水域近接除外（マイニング除外）」は不採用。滑走路という物理的制約（直線・一定長）に直結する条件のみ追加し、false negativeを増やすリスクのある条件は見送った。

**注意：** Sentinel-2は依然として「滑走路特異的な識別」はできず、伐採・マイニング等の誤候補が混入しうる。最終確認はNICFI（YOLO形状判定）が担うため、誤候補は`active`に格上げされず6ヶ月で`inactive`に自動収束する2段構成が前提（ARCHITECTURE.md Models参照）。

### パラメータ調整の方針（確定）

初期パラメータ（NDVI閾値・アスペクト比・直線性残差・最小長さ）は**意図的に緩め（広く拾う方向）**に設定し、再現率の確保はこの初期設計判断に委ねる。スタッフ確認結果（正誤ラベル）は、TPの変化量分布にフィットさせ、**分布上で信頼度99.9%をカバーする水準まで外挿的に閾値を決定**する（単純な「観測されたTPの最小値」ではなく、まだ観測データに含まれない微小変化量のTPも一定確率で拾えるようにする）。この方式のため、再計算の結果は必ずしも「引き締め」になるとは限らず、緩くなることもあり得る。

理由：見逃し（False Negative）を定量検証するための網羅的な正解データは作れない。既知滑走路178枚（前任者モデルが明確に検出できた事例）を再現率検証に使うことも検討したが、これは「分かりやすい滑走路」という選択バイアスがかかった集合であり、Sentinel-2が見逃しやすい境界的なケース（建設初期・伐開が小さい・雲の影響等）を代表しない。そのため178枚で見逃しがなかったとしても、フィルタが十分緩いことの証明にはならず、不採用とした。

**サンプルサイズの留意点：** TP件数が少ない場合（数十〜百件程度）、経験分布から99.9パーセンタイル（下側0.1%点）を直接読み取るとテール推定が不安定になる。件数が十分でなければ対数正規分布等を仮定したパラメトリックな推定を検討する。

## Global Forest Watch（GFW）連携と付加特徴量（確定・2026-07-01追記）

スタッフとの協議により、以下の3点を追加で組み込む方針を確定した。

### GFWをSentinel-2と並行する早期警戒ソースとして追加

[Global Forest Watch](https://www.globalforestwatch.org/)のIntegrated Deforestation Alerts（GLAD-L / GLAD-S2 / RADD / DIST-ALERTの統合）を、Sentinel-2変化検出と並行する**第2の早期警戒ソース**として追加する。

**GFWの内実（解像度・更新頻度は統一されていない）：**

| サブシステム | 衛星 | 解像度 | 更新頻度 |
|---|---|---|---|
| GLAD-L | Landsat（光学） | 30m | 週次 |
| GLAD-S2 | Sentinel-2（光学） | 10m | 週次（雲の影響あり） |
| RADD | Sentinel-1（SARレーダー） | 10m | ほぼリアルタイム（雲の影響なし、熱帯では6〜12日で欠測なしのカバレッジ） |

どのサブシステムが検知したかで実際の解像度・鮮度が変わるため、GFW検出結果にはどのサブシステム由来かのタグを保持し、将来的にGFW較正曲線の特徴量に含めることを検討する。

- **役割：** GFWも滑走路の形状確認はできないため、Sentinel-2と同じ「早期警戒のみ、`active`への格上げは不可」の役割とする。`active`格上げの権限はNICFIのみという既存のドミナンスルールは変更しない。
- **信頼度：** Sentinel-2の較正曲線（NDVI変化量マージンを特徴量とする）はGFWにはそのまま適用できない（特徴量の定義が異なるため）。GFW自身が提供する特徴量（confidenceレベル：low/nominal/highの順序値等）を使い、**GFW専用の較正曲線を別途構築**する。
- **Sentinel-2×GFW同時検出時の信頼度合成：** 過去にNICFI×Sentinel-2間の「corroboration boost」は「適切な確率モデルなしに恣意的になる」という理由で不採用としたが、Sentinel-2×GFWは共に非確証ソース同士であり、両者が検出した候補の正誤ラベルが揃えば、`P(TP | Sentinel-2マージン, GFW特徴量)`をロジスティック回帰等で統計的に推定できる。恣意的なブースト値ではなく、データに基づく合成が可能な場合に限り採用する（詳細は下記「構築フロー」参照）。
- **将来の拡張性：** Coca Crop予測等の将来機能でもGFWデータは利用価値があるため、GFW APIクライアントは汎用モジュールとして実装する。

**なぜ自前でSentinel-1（SAR）解析を作らず、GFW/RADDという既成アラートを使うか：**

Sentinel-1データ自体はRADDと同じくCC BY 4.0で無料公開されているが、以下の理由から現時点では自前実装ではなくGFW/RADDの既成アラートを採用する。

1. RADDの実質的な更新頻度（熱帯で6〜12日）はSentinel-2晴天時（5日）と比べて劇的に速いわけではなく、優位性は主に「雲の影響を受けない」点に限られる
2. SARデータ特有のスペックルノイズ除去・地形起因の歪み補正は、光学画像のNDVI変化検出より専門性が高く、実装・検証コストが増す
3. RADD/GLADは既に完成品のアラート（点＋confidenceの順序値）であり、Sentinel-2で行っているような「自前の連続値特徴量＋滑走路形状フィルタ（アスペクト比・直線性・最小長さ）＋TP分布からの閾値外挿」という較正手法をそのまま適用できない

**将来検討事項（自前Sentinel-1レイヤー）：** 将来的に自前でSentinel-1に対して滑走路形状フィルタを適用する独自レイヤーを構築できれば、「雲の影響を受けない」かつ「滑走路形状に最適化されている」の両方を満たし、現状のSentinel-2（形状最適化だが雲に弱い）・GFW（雲に強いが汎用的な森林減少検出）のいずれよりも精度の高い早期警戒層になり得る。ただしSARの形状認識はスペックルノイズにより光学より難しいため、NICFIによる最終形状確認という2段構成の必要性自体は将来も残ると考えられる。今回は着手せず、数年後の再学習サイクルでの拡張候補として記録するに留める。

### Sentinel-1（SAR）を単独モデルでなく変化検出の特徴量として統合（追加・2026-07-01）

チームメンバーの提案：滑走路部分は平滑な裸地のため、SARでは後方散乱が弱く暗く写る特性がある。この特性は「独立した検出モデル」としてではなく、**Sentinel-2変化検出の追加特徴量**として組み込む方向で検討する。理由はGFWの場合と同様（Sentinel-1単独では10m解像度のため形状による独立判定が困難、既存のRADDと役割が重複）だが、暗く写るという特性値自体は既存のNDVIマージンと相関する可能性があり、情報として捨てるのはもったいない。

- Sentinel-2の各候補地点・日時に対応するSentinel-1後方散乱値（バックスキャッター強度の低下）を追加特徴量として取得
- ステップ2の正誤ラベルが揃った時点で、NDVIマージン単独よりも予測力が上がるか統計的に検証
- 相関が確認できれば、GFWのケースと同じ枠組み（`P(TP | Sentinel-2マージン, SAR特徴量)`のロジスティック回帰等）でNDVIマージンと組み合わせた較正モデルに拡張
- 相関が確認できなければ、Slope・距離データと同様「メタデータとして保持のみ、組み込みは見送り」とする

### Slope・河川/村からの距離データ（付加特徴量として保持）

現場スタッフの経験則：
- 違法滑走路は河川・道路・既存の村落に近い場所に多い
- 土砂崩れは傾斜地（Slope）でのみ発生し、違法滑走路は平地にのみ存在するため、両者の誤認識の切り分けにSlope情報が有効

これらは**事前の除外フィルタとしては採用しない**（CONCEPT_NOTEの既存方針「孤立度」「水域近接除外」を見送った理由＝False Negativeリスクと同じ理由）。代わりに、Slope・河川距離・村落距離をCandidateごとのメタデータとして保持しておき、スタッフ確認によるTrue/Falseラベルが蓄積された後、**統計的に有意な相関が確認できた場合に限り**信頼度算出ロジックに組み込むことを検討する。相関が確認できなければ組み込まない（保持しておくこと自体はコストが低いため、判断を先送りできる）。

### 構築フロー（改訂・2026-07-01）

目的別に2ステップに再整理した。ステップ2は「早期警戒ロジック（Sentinel-2・GFW）自体の較正」、ステップ3は「NICFI確証によるYOLO学習データ収集」であり、ステップ3bのみステップ2完了が前提。日付は具体例、実際の取得日はデータ取得時に確定する。

**画像取得**
- Sentinel-2：本番運用と同じ7日間隔の週次ペアで取得（例：2025-06-01 / 2025-06-08）
- Planet NICFI：月次モザイク1点（例：2025-06-15）
- GFW：Sentinel-2と同一期間のアラートをAPI取得

**ステップ2：早期警戒ロジックの較正（NICFI不使用、計700枚）**

| サブステップ | 内容 | 枚数 | 目的 |
|---|---|---|---|
| 2a | 2025-06-01/06-08ペアでNDVI変化検出（NDVI変化量・細長形状・直線性・最小長さの4条件を緩めに設定）→ スタッフ正誤判定 | 500枚 | Sentinel-2閾値の外挿決定（TP変化量分布から99.9%信頼度） |
| 2b | GFWは検出したがSentinel-2の候補生成では拾えなかった差分地点 → スタッフ正誤判定 | 100枚 | Sentinel-2単独では見逃すFalse Negativeの回収効果を検証 |
| 2c | Sentinel-2は検出したがGFWでは検出されなかった地点を意図的に抽出 → スタッフ正誤判定 | 100枚 | Sentinel-2×GFW同時検出時の信頼度ブーストの統計的検証（2aのGFW重複分と比較） |

- **既知の限界（2aに関して）：** クリアリング後に部分緑化が進むケースでは、間隔が長い（雲でスキップした）場合の変化量が週次より小さく出ることがあり、週次最適化パラメータでは拾いきれない可能性がある。今回は解決せず既知の限界として記録する
- **2b・2cのサンプリング方法：** 2aの500枚は無作為抽出のまま行う。GFWとの自然な重複率が極端に高い（例：500枚中480枚が重複）場合、2aだけでは「Sentinel-2単独・GFW非該当」のケースが少数（例：20枚程度）しか含まれず、ブースト効果の統計的検証には不十分なため、2cとして別途意図的に抽出する。2aの閾値較正計算自体は無作為抽出のまま歪めず、2b・2cは別集計として扱う

**ステップ3：YOLO学習データ収集（NICFI使用、計1000枚。3a・3bは並行実施可能、3bは2.完了が前提）**

| サブステップ | 内容 | 枚数 |
|---|---|---|
| 3a | NICFI（2025-06-15）に既存Faster R-CNNを閾値なしで適用 → スタッフ正誤判定 | 500枚 |
| 3b | 2aのロジックを次の週次ペア（2025-06-08/06-15）に適用 → 同一地点のNICFI画像でスタッフ正誤判定 | 500枚 |

- 元178枚＋3a 500枚＋3b 500枚＝**計1178枚**でYOLO新規学習
- 3b由来のサンプルには「Sentinelトリガー由来」のタグを付与（信頼度算出ではなく、将来のデータ再評価・再学習サイクルでのトレーサビリティ目的）

**ステップ3c：マルチテンポラル拡張（追加・2026-07-01、2026-07-01訂正）**

対象：元178枚の地点＋ステップ2aでTRUE確認された地点（件数は2a実施後に確定、目安100地点程度）。これらの地点について、同じ座標の**3ヶ月前・6ヶ月前・9ヶ月前**のNICFIモザイク（地点あたり3枚追加＝計4倍）を追加取得する。

- **自動でPositiveラベルは付けない。** 元の時点で真陽性と確認された地点であっても、3/6/9ヶ月前の時点では滑走路が未建設・植生に覆われて判別不能等の理由でFalseになりうるため、追加取得した画像は**通常のステップ2/3と同じスタッフT/Fレビューに回す**（軽量チェックではなくフル判定）
- **地点単位でtrain/val分割する。** 同一地点の複数時期画像が無作為にtrain/valへ分かれると、実質的に同じ場所を学習・検証双方で使うことになり、汎化性能の検証が汚染される
- 3/6/9ヶ月前という固定間隔にしたのは、もともとの学習対象（178枚・ステップ3a/3b）が比較的新しい時期の写真を想定していたため、そこから遡って時間軸のバリエーションを追加する狙い。季節・雲条件による多様性は狙いとして副次的に得られるが、選定基準は「固定間隔」であり任意選定ではない

**ステップ4：パラメータ再計算＋可変信頼度の較正**
- 2aの500枚＋3bの500枚（計1000枚、NICFI裏取り済みでラベル信頼性は同格として扱う）をプールし、99.9%信頼度でSentinel-2閾値を再計算。2a単独より緩くなる可能性がある
- 各候補の「閾値からのマージン（主にNDVI変化量）」と正誤ラベルの関係を単調な較正曲線（アイソトニック回帰等）でフィットし、Sentinel-2検出ごとの可変信頼度を算出できるようにする（固定の平均精度値は使わない）
- 2bのデータを使い、GFW自身の特徴量に基づく較正曲線を別途構築する
- 2a（GFW重複分）と2cを比較し、Sentinel-2×GFW同時検出の信頼度ブーストが統計的に正当化できるか検証する（`P(TP | Sentinel-2マージン, GFW特徴量)`のロジスティック回帰等）。正当化できない場合はブーストを採用しない
- **反復は1回で打ち切る：** さらに閾値を緩めて再度候補を集め直す、という反復は費用対効果が急速に悪化するため行わない。次の見直しは数年後の再学習サイクルに委ねる

**レビュー対象の総枚数：ステップ2（700枚）＋ステップ3（1000枚）＝ 1700枚。**ステップ3cのマルチテンポラル拡張（約278地点×3ヶ月分＝目安834枚、通常のT/Fレビュー）は上記1700枚とは別枠としてカウントする
